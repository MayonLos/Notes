## C++ 内存模型

| 区域 | 分配方式 | 生命周期 |
|:---|:---|:---|
| **栈** | 自动 | 作用域结束时销毁 |
| **堆** | 手动 `new/delete` | 显式释放（用智能指针管理） |
| **静态段** | 编译期 | 整个程序运行期间 |

## 智能指针

### `unique_ptr` — 独占所有权（首选）

```cpp
auto ptr = std::make_unique<int>(42);  // 推荐 make_unique
auto ptr2 = std::move(ptr);            // 可移动，ptr 变 nullptr
// 离开作用域自动释放，零额外开销
```

### `shared_ptr` — 共享所有权（引用计数）

```cpp
auto p1 = std::make_shared<int>(42);
auto p2 = p1;               // 引用计数 = 2
std::cout << p1.use_count();
// 最后一个 shared_ptr 销毁时释放
```

### `weak_ptr` — 弱引用（打破循环引用）

```cpp
class Node {
public:
    std::shared_ptr<Node> next;
    std::weak_ptr<Node> prev;    // 不增加引用计数
};
```

> [!warning] 循环引用内存泄漏
> 两个 `shared_ptr` 互相引用，引用计数永远不为 0。用 `weak_ptr` 打破循环。

> [!warning] `new/delete` 不匹配
> `new` 配 `delete`，`new[]` 配 `delete[]`，混用导致 UB。**现代 C++ 应完全避免裸 `new/delete`**。

## RAII 原则

资源在构造时获取，在析构时释放，即使发生异常也能安全清理：

```cpp
class File {
    FILE* handle;
public:
    File(const std::string& path) { handle = fopen(path.c_str(), "r"); }
    ~File() { if (handle) fclose(handle); }
    File(const File&) = delete;
    File& operator=(const File&) = delete;
};
{ File f("data.txt"); }   // 离开作用域自动关闭
```

## Move 语义（C++11）

```cpp
// Move 构造：接管资源，不复制
class MyVec {
    int* data; int size;
public:
    MyVec(MyVec&& other) noexcept
        : data(other.data), size(other.size) {
        other.data = nullptr; other.size = 0;
    }
};

std::string s = "hello";
vec.push_back(std::move(s));   // 移动而非拷贝，s 变空
```

## 现代 C++ 特性

```cpp
// auto 类型推导
auto x = 42;
auto it = vec.begin();

// Lambda 表达式
auto add = [](int a, int b) { return a + b; };
int mult = 3;
auto multiply = [mult](int x) { return x * mult; };  // 值捕获

// constexpr 编译期计算
constexpr int factorial(int n) {
    return n <= 1 ? 1 : n * factorial(n - 1);
}
constexpr int f5 = factorial(5);  // 编译期得到 120
```

## 最佳实践

| 原则 | 说明 |
|:---|:---|
| 优先 `unique_ptr` | 独占所有权时首选，零开销 |
| 用 `make_unique/make_shared` | 避免裸 `new`，异常安全 |
| 遵循 RAII | 资源在析构函数中释放 |
| 大对象用 Move | `std::move` 避免拷贝开销 |
