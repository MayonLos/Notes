> **一句话定义**：以对象为中心的编程范式，核心四大原则为抽象、封装、继承、多态，是构建大规模可维护程序的基础。

## 四大原则

| 原则 | 含义 |
|:---|:---|
| **抽象** | 隐藏实现细节，暴露本质特征 |
| **封装** | 隐藏内部状态，提供公共接口 |
| **继承** | 代码复用，建立类层次结构 |
| **多态** | 统一接口，多样实现 |

## 类与对象

### 基本结构

```cpp
class Animal {
public:
    std::string name;
    void eat() { std::cout << name << " 在吃\n"; }

protected:
    int age;        // 子类可访问，外部不可

private:
    int energy;     // 仅类内可访问
};

Animal dog;
dog.name = "小黑";
dog.eat();
```

### 构造函数与析构函数

```cpp
class Person {
private:
    std::string name;
    int age;
public:
    Person() : name("未知"), age(0) {}              // 默认构造
    Person(const std::string& n, int a) : name(n), age(a) {}  // 参数化构造
    ~Person() { std::cout << "Person " << name << " 销毁\n"; }
};
```

> [!warning] 初始化列表顺序
> 初始化列表的执行顺序遵循**成员声明顺序**，而非初始化列表中的书写顺序。两者不一致时容易产生逻辑错误。

## 访问修饰符

| 修饰符 | 类内 | 派生类 | 类外 |
|:---|:---|:---|:---|
| `public` | ✅ | ✅ | ✅ |
| `protected` | ✅ | ✅ | ❌ |
| `private` | ✅ | ❌ | ❌ |

## 继承与多态

### 继承

```cpp
class Vehicle {
public:
    virtual void start() { std::cout << "车启动\n"; }
    virtual ~Vehicle() {}    // 基类析构函数必须为虚函数！
};

class Car : public Vehicle {
public:
    void start() override {  // override 关键字（推荐）
        std::cout << "汽车引擎启动\n";
    }
};
```

> [!warning] 基类必须有虚析构函数
> 通过基类指针删除派生类对象时，若析构函数非虚，只会调用基类析构函数，导致内存泄漏。

### 多态（动态绑定）

```cpp
void test(Vehicle& v) {
    v.start();    // 运行时根据实际类型调用
}

Car car;
test(car);        // 输出"汽车引擎启动"
```

**虚函数机制**：编译期建立虚表（vtable），运行期通过 vptr 查表调用实际函数，开销极小（一次指针间接）。

## 抽象类与接口

```cpp
class Shape {
public:
    virtual void draw() = 0;          // 纯虚函数
    virtual double area() const = 0;
    virtual ~Shape() {}
};

class Circle : public Shape {
    double radius;
public:
    void draw() override { std::cout << "画圆\n"; }
    double area() const override { return 3.14159 * radius * radius; }
};

// Shape s;  // 错误：不能实例化抽象类
Circle c;    // 正确
```

## 运算符重载

```cpp
class Vector2D {
    double x, y;
public:
    Vector2D operator+(const Vector2D& o) const {
        return Vector2D(x + o.x, y + o.y);
    }
    bool operator==(const Vector2D& o) const {
        return x == o.x && y == o.y;
    }
};
```

> [!warning] 运算符重载陷阱
> - `&&`、`||` 重载后失去短路特性，不建议重载
> - `=`、`[]`、`()` 只能作成员函数
> - `<<`、`>>` 通常作友元或非成员函数
