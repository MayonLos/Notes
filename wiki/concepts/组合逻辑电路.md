> [!tip] 学习路径
> **前置**：[[原码反码补码]]（理解数值的二进制表示，加法器意义才清晰）
> **本页**：组合逻辑——逻辑门→加法器→MUX/DEMUX→编码/译码→竞争与冒险
> **后续**：→ [[锁存器和触发器]]（触发器、寄存器、计数器）

## 一、基本逻辑门

![[logic_gates.png]]

| 门名称 | 逻辑表达式 | 真值描述 |
|:---|:---|:---|
| **AND（与）** | `F = A·B` | 全1出1，有0出0 |
| **OR（或）** | `F = A+B` | 有1出1，全0出0 |
| **NOT（非）** | `F = Ā` | 取反 |
| **NAND（与非）** | `F = A·B‾` | AND后取反，**通用门** |
| **NOR（或非）** | `F = A+B‾` | OR后取反，**通用门** |
| **XOR（异或）** | `F = A⊕B` | 相同出0，相异出1 |
| **XNOR（同或）** | `F = A⊙B` | 相同出1，相异出0 |

> [!tip] 通用门
> **NAND 和 NOR 是通用门**：仅用其中一种即可实现任意逻辑函数。实际 IC 设计中常用 NAND-NAND 两级结构。

### 布尔代数基础

| 定律 | 表达式 |
|:---|:---|
| **交换律** | `A+B = B+A`，`A·B = B·A` |
| **分配律** | `A·(B+C) = A·B + A·C` |
| **吸收律** | `A+A·B = A` |
| **德摩根定律** | `A·B‾ = Ā+B̄`，`A+B‾ = Ā·B̄` |
| **互补律** | `A+Ā = 1`，`A·Ā = 0` |

$$\overline{A \cdot B} = \bar{A} + \bar{B} \qquad \overline{A + B} = \bar{A} \cdot \bar{B}$$

> [!important] 德摩根口诀
> 取反线断开，**与变或、或变与**，各变量取反。

### 卡诺图化简（K-Map）

![[karnaugh_map.png]]

- 每组必须是 **1、2、4、8……** 个相邻1（2的幂次）
- 分组尽量**越大越好**（消去更多变量）
- 允许**重叠**和**卷边**（左右、上下边界视为相邻）
- 所有的1必须被覆盖

```
设计流程：需求分析 → 定义输入/输出 → 列真值表 → 写逻辑表达式 → 卡诺图化简 → 画电路图
```

## 二、加法器

> 加法器是最基本的算术运算电路，是 ALU 的核心组件。

### 2.1 半加器（Half Adder）

**功能**：只计算两个1位二进制数相加，**不处理进位输入**。

![[half_adder.png]]

| 输出 | 逻辑表达式 |
|:---|:---|
| 和位 S | `S = A ⊕ B` |
| 进位 Cout | `Cout = A · B` |

> [!warning] 局限性
> 半加器**不能处理来自低位的进位**，只能用于最低位（LSB）的加法。

### 2.2 一位全加器（Full Adder）

**功能**：增加**进位输入 Cin**，可以级联实现多位加法。

![[full_adder.png]]

| 输出 | 逻辑表达式 |
|:---|:---|
| 和位 S | `S = A ⊕ B ⊕ Cin` |
| 进位 Cout | `Cout = A·B + (A⊕B)·Cin` |

> [!note] 关键理解
> - **S** = 三个输入的奇偶校验（1的个数为奇数则S=1）
> - **Cout** = 三输入中至少有两个为1

### 2.3 多位全加器

**行波进位加法器**：N 个全加器串联，低位 Cout → 高位 Cin。

![[multi_bit_adder.png]]

| 改进方案 | 原理 | 延迟 |
|:---|:---|:---|
| **超前进位**（Carry Look-ahead） | 并行计算所有进位 | O(log N) |
| **进位选择**（Carry Select） | 预计算两种进位结果按实际选择 | O(√N) |

> [!warning] 行波进位延迟
> 延迟随位数线性增长：$t_{delay} = N \times t_{carry}$，高速场景必须改进。

## 三、数据选择器 MUX

> **功能**：从多路输入中**选择**一路输出，类比"多路开关"。

### 4选1 MUX

![[mux_4to1.png]]

$$Y = \bar{S_1}\bar{S_0}D_0 + \bar{S_1}S_0 D_1 + S_1\bar{S_0}D_2 + S_1 S_0 D_3$$

| S1 | S0 | 输出 Y |
|:---:|:---:|:---:|
| 0 | 0 | D0 |
| 0 | 1 | D1 |
| 1 | 0 | D2 |
| 1 | 1 | D3 |

> [!note] MUX 实现任意逻辑函数
> MUX 的逻辑表达式是**最小项之和**，可将真值表值直接接入数据端实现任意函数。

### 8选1 MUX

![[mux_8to1.png]]

- 数据输入：D0~D7（8路），选择控制：S0~S2（3位）
- 常见芯片：**74LS151**（含互补输出），**74LS153**（双 4-to-1 MUX）

## 四、多路分配器 DEMUX

> **功能**：将单路输入分配到多路输出的某一路，类比"单刀多掷开关"。

### 4路 DEMUX

![[demux_1to4.png]]

| S1 | S0 | 输出 |
|:---:|:---:|:---|
| 0 | 0 | Y₀ = D，其余=0 |
| 0 | 1 | Y₁ = D，其余=0 |
| 1 | 0 | Y₂ = D，其余=0 |
| 1 | 1 | Y₃ = D，其余=0 |

> [!important] DEMUX = 译码器 + 数据输入
> 将译码器的使能端 EN 改为数据输入端 D，即得 DEMUX。

## 五、编码器

> **功能**：将 $2^n$ 条输入线编码为 $n$ 位二进制输出。

### 基本编码器 vs 优先编码器

| | 基本编码器 | 优先编码器 |
|:---|:---|:---|
| 输入约束 | 同时只能有1个输入有效 | 允许多个输入同时有效 |
| 冲突处理 | 多输入有效→输出不确定 | 响应**优先级最高**的输入 |

### 8线-3线 优先编码器

![[encoder_8to3.png]]

| 优先级 | 输入 | 输出 Y₂Y₁Y₀ |
|:---:|:---:|:---:|
| 最高 | I₇ | 111 |
| ↓ | I₆ | 110 |
| ↓ | I₀ | 000 |（最低）

- **GS**（Got Something）：至少有一个输入有效，用于区分"I₀=1"和"无输入"
- **EO**（Enable Out）：无任何输入有效，用于级联扩展

**常见芯片**：74LS148（8-to-3 优先编码器）

## 六、译码器

> **功能**：将 n 位二进制输入译为 $2^n$ 条互斥输出线中的某一条。

### 2线-4线 译码器

![[decoder_2to4.png]]

$$Y_0 = \bar{A_1}\bar{A_0}, \quad Y_1 = \bar{A_1}A_0, \quad Y_2 = A_1\bar{A_0}, \quad Y_3 = A_1 A_0$$

> [!note] 本质
> 每个输出对应一个**最小项**，n-to-2ⁿ 译码器产生 n 个变量的**所有最小项**。

### 3线-8线 译码器

![[decoder_3to8.png]]

**常见芯片**：74LS138（含3个使能端，方便级联）

级联扩展：最高位 + 反相器控制两个 74LS138 的使能端，实现 4-to-16。

> [!tip] 译码器 = 最小项发生器
> 译码器 + OR 门 = 任意组合逻辑函数

### 七段译码显示器

![[seven_segment.png]]

七段数码管（a~g 段）通过控制各段亮灭显示数字。

| 类型 | 公共端 | 亮灯条件 | 常用译码器 |
|:---|:---:|:---|:---|
| **共阴极** | 接地 | 段驱动 = 1 | 74LS48 |
| **共阳极** | 接 VCC | 段驱动 = 0 | 74LS47 |

## 七、数值比较器

> **功能**：比较两个二进制数大小，输出 `A>B`、`A=B`、`A<B` 三种状态。

### 1位比较器

![[comparator_1bit.png]]

| 输出 | 逻辑表达式 |
|:---|:---|
| A > B | `A · B̄` |
| A = B | `A ⊙ B`（同或） |
| A < B | `Ā · B` |

### 多位比较器

![[comparator_multibit.png]]

**算法**：从 MSB（最高位）向 LSB 逐位比较，首个不等的位决定结果。

$$EQ = (A_3 \odot B_3)(A_2 \odot B_2)(A_1 \odot B_1)(A_0 \odot B_0)$$

**常见芯片**：74LS85（4位，含级联输入端）

## 八、竞争与冒险

> 由于不同路径**信号传播延迟不同**，输入变化瞬间输出端可能出现短暂错误脉冲（毛刺/Glitch）。

![[hazard_timing.png]]

### 基本概念

- **竞争（Race）**：多个信号经不同路径到达同一点，到达时间有先后
- **冒险（Hazard）**：竞争导致输出出现**短暂的错误脉冲**

> [!important] 关系
> **竞争**是原因，**冒险**是结果。有竞争不一定有冒险，但有冒险一定有竞争。

### 冒险分类

| 类型 | 描述 |
|:---|:---|
| **静态1冒险** | 输出应稳定为1，却出现短暂的0脉冲 |
| **静态0冒险** | 输出应稳定为0，却出现短暂的1脉冲 |
| **动态冒险** | 输出应变化一次，却多次跳变后才稳定 |

**卡诺图中的冒险**：两个相邻但不重叠的分组，输入变化跨越两组时可能出现冒险。

### 消除方法

| 方法 | 原理 | 适用场景 |
|:---|:---|:---|
| **增加冗余项** | 在相邻分组间添加覆盖两者的冗余乘积项 | FPGA / 定制逻辑（推荐） |
| **滤波电容** | 输出端加小电容滤除窄脉冲 | 低速、非关键路径 |
| **时钟采样** | 只在时钟有效沿采样，忽略过渡期毛刺 | 同步时序电路（标准方案） |

示例：`F = AB + ĀC` → 加冗余项后 `F = AB + ĀC + BC`（BC 为冗余项）

## 九、基于 MSI 的组合逻辑设计

> **MSI 设计思路**：利用中规模集成电路（MUX、译码器、加法器）的固有逻辑结构，通过**配置其数据/控制端**实现任意组合逻辑函数。

### 9.1 用 MUX 实现任意逻辑函数

**原理**：n 位地址输入的 MUX 输出是 $2^n$ 个最小项的一种选择求和。将函数的真值表值接入数据端即可。

#### 例：用 8选1 MUX (74HC151) 实现 $F(A,B,C)=\sum m(1,3,5,6)$

| 地址 ABC | D 端接法 | 说明 |
|:---|:---|:---|
| 000 | D₀=0 | m₀ 不在 Σm 中 |
| 001 | D₁=1 | m₁ 在 Σm 中 |
| 010 | D₂=0 | m₂ 不在 Σm 中 |
| 011 | D₃=1 | m₃ 在 Σm 中 |
| 100 | D₄=0 | m₄ 不在 Σm 中 |
| 101 | D₅=1 | m₅ 在 Σm 中 |
| 110 | D₆=1 | m₆ 在 Σm 中 |
| 111 | D₇=0 | m₇ 不在 Σm 中 |

> 将 A, B, C 分别接 MUX 地址端 S₂, S₁, S₀，数据端 D₀~D₇ 按上表接 VCC/GND 即可。

#### 降维法：用 $2^{n-1}$ MUX 实现 $n$ 变量函数

将函数的一个变量（通常是最高位）用作数据输入，其余 $n-1$ 变量接 MUX 地址。每个数据端接入的是剩余变量的**残留函数**（0/1/原变量/反变量）。

**常见芯片**：
| 芯片 | 规格 | 特点 |
|:---|:---|:---|
| 74HC151 / CT74LS151 | 8选1 | 有互补输出 Y 和 Ȳ |
| 74HC153 / CT74LS153 | 双4选1 | 两个独立 4-to-1 MUX，共享地址 |
| 74HC150 | 16选1 | 4 位地址 |

### 9.2 用译码器实现任意逻辑函数

**原理**：n-to-2ⁿ 译码器的每个输出对应一个最小项 $\overline{m_i}$（低有效）。将函数所需最小项对应的输出端接入 **NAND 门**，输出即为该函数。

#### 例：用 3-8 译码器 (74HC138) 实现 $F(A,B,C)=\sum m(2,4,6,7)$

$$
F = \overline{Y_2 \cdot Y_4 \cdot Y_6 \cdot Y_7}
$$

将 74HC138 的 Ȳ₂, Ȳ₄, Ȳ₆, Ȳ₇ 接入一个 4 输入 NAND 门 → 输出即 F。

> [!tip] 译码器 + NAND = 任意组合逻辑
> 低有效译码器天然输出最小项的非。用 NAND 门求和（德摩根等效 OR），极简实现。

### 9.3 Design Flow 总结

```
SSI 设计：真值表 → 卡诺图化简 → 门级电路
MSI 设计：真值表 → 选器件(芯片) → 配置数据/地址端 → 芯片+少量门
```

| 方法 | 额外门数 | 适用场景 |
|:---|:---|:---|
| MUX 直连法 | 0 | 变量数 ≤ MUX 地址数 |
| MUX 降维法 | 少量 | 变量数 = MUX 地址数 + 1 |
| 译码器 + NAND | 1 个 NAND 门 | 任意 n ≤ 译码器输入数 |

## 十、加法器综合应用

### 10.1 四位可控加减法器

用 **74LS283（4 位超前进位加法器）** + XOR 门实现加减法切换：

- 加法模式（M=0）：$B_i$ 直通 → $S = A + B$
- 减法模式（M=1）：$B_i$ 取反 + $C_0=1$ → $S = A - B$（补码减法）

```
关键：M 接每个 B_i 的 XOR 输入和 C₀，实现 B̄+1（即 -B 的补码）
```

### 10.2 8421 码 ↔ 余 3 码转换

**8421 → 余3**：$余3 = 8421 + 3$。用 74LS283 将 8421BCD 输入 + $(0011)_2$。

**余3 → 8421**：$8421 = 余3 - 3 = 余3 + 1101_2$（-3 的补码，4位）。同用 74LS283，B 端接 $1101$，$C_0=0$。
