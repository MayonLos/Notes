> **一句话定义**：现代 C++（C++11+）通过智能指针、Move 语义和 RAII 原则自动管理内存，消除手动 `new/delete` 引入的内存泄漏和悬垂指针问题。

## C++ 内存模型

| 区域 | 分配方式 | 生命周期 |
|:---|:---|:---|
| **栈** | 自动 | 作用域结束时销毁 |
| **堆** | 手动 `new/delete` | 显式释放（用智能指针管理） |
| **静态段** | 编译期 | 整个程序运行期间 |

## 智能指针

### `unique_ptr` — 独占所有权（首选）

```cpp
auto ptr = std::make_unique<int>(42);  // 推荐 make_unique
auto ptr2 = std::move(ptr);            // 可移动，ptr 变 nullptr
// 离开作用域自动释放，零额外开销
```

### `shared_ptr` — 共享所有权（引用计数）

```cpp
auto p1 = std::make_shared<int>(42);
auto p2 = p1;               // 引用计数 = 2
// 最后一个 shared_ptr 销毁时释放
```

### `weak_ptr` — 弱引用（打破循环引用）

```cpp
class Node {
public:
    std::shared_ptr<Node> next;
    std::weak_ptr<Node> prev;    // 不增加引用计数
};
```

> [!warning] 循环引用内存泄漏
> 两个 `shared_ptr` 互相引用，引用计数永远不为 0。用 `weak_ptr` 打破循环。

> [!warning] `new/delete` 不匹配
> `new` 配 `delete`，`new[]` 配 `delete[]`，混用导致 UB。**现代 C++ 应完全避免裸 `new/delete`**。

## RAII 原则

资源在构造时获取，在析构时释放，即使发生异常也能安全清理：

```cpp
class File {
    FILE* handle;
public:
    File(const std::string& path) { handle = fopen(path.c_str(), "r"); }
    ~File() { if (handle) fclose(handle); }
    File(const File&) = delete;
    File& operator=(const File&) = delete;
};
// { File f("data.txt"); }  离开作用域自动关闭
```

## Move 语义（C++11）

```cpp
std::vector<int> a = {1, 2, 3};
std::vector<int> b = std::move(a);  // 移动：a 变为空，O(1)
// 拷贝是 O(n)，移动是 O(1)——转移所有权而非复制数据
```

> [!important] 何时使用 move
> 当对象不再需要后，用 `std::move` 将其资源转移给另一个对象，避免不必要的深拷贝。

## 常见内存问题

| 问题 | 描述 | 预防 |
|:---|:---|:---|
| 内存泄漏 | 堆内存没有释放 | 智能指针 |
| 悬垂指针 | 指向已释放内存 | 智能指针/RAII |
| 野指针 | 未初始化的指针 | 初始化为 nullptr |
| 双重释放 | delete 两次 | `unique_ptr` |
| 数组越界 | 超出合法索引 | 使用 `at()` 或 `vector` |
